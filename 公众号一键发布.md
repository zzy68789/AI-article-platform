# 公众号一键发布

## 功能概览

本功能基于微信公众号官方接口。除平台管理员可继续使用平台默认公众号外，普通用户可以通过微信开放平台第三方平台扫码授权自己的公众号，并在文章详情页选择目标公众号保存草稿或提交发布。

核心链路：

```text
用户扫码授权公众号 -> 获取 authorizer_access_token -> 文章详情页选择公众号 -> 上传图片 -> draft/add -> freepublish/submit -> freepublish/get
```

## 已实现内容

- 新增微信发布配置：`wechat.app-id`、`wechat.app-secret`、`wechat.default-author`、接口 Base URL 与超时时间。
- 新增 `wechat_publish_record` 表，用于记录草稿 mediaId、发布 publishId、官方状态、文章链接和失败原因。
- 新增后端接口：
  - `POST /api/article/wechat/draft/{taskId}`：保存到公众号草稿箱。
  - `POST /api/article/wechat/publish/{taskId}`：创建草稿并提交发布。
  - `GET /api/article/wechat/status/{taskId}`：查询本地最新发布记录。
  - `GET /api/article/wechat/official-status/{publishId}`：查询微信官方发布状态并回写本地记录。
- 新增 Markdown 渲染服务，将文章 Markdown 转为带基础公众号样式的 HTML，并将正文图片上传到微信。
- 前端文章详情页新增发布按钮、发布状态展示和官方状态刷新按钮。
- 新增公众号授权管理页 `/wechat/accounts`，支持扫码授权、设置默认公众号和解除绑定。
- 授权公众号的 `authorizer_refresh_token` 使用 AES-GCM 加密后存储，普通用户只能使用自己绑定的公众号。
- README 和 `.env.example` 已补充微信发布配置说明。

## 数据库变更

执行增量脚本：

```bash
mysql -uroot -p < sql/add_wechat_publish_record.sql
mysql -uroot -p < sql/add_wechat_open_platform.sql
```

第二个脚本会创建授权公众号、第三方平台 ticket、授权临时状态表，并让发布记录关联目标公众号。

## 配置方式

在本地或部署环境配置以下变量：

```bash
WECHAT_APP_ID=你的公众号AppID
WECHAT_APP_SECRET=你的公众号AppSecret
WECHAT_DEFAULT_AUTHOR=默认作者
WECHAT_BASE_URL=https://api.weixin.qq.com/cgi-bin
WECHAT_CONNECT_TIMEOUT_MS=10000
WECHAT_READ_TIMEOUT_MS=30000

WECHAT_OPEN_PLATFORM_ENABLED=true
WECHAT_COMPONENT_APP_ID=第三方平台AppID
WECHAT_COMPONENT_APP_SECRET=第三方平台AppSecret
WECHAT_COMPONENT_TOKEN=第三方平台消息校验Token
WECHAT_COMPONENT_AES_KEY=第三方平台消息加解密Key
WECHAT_AUTHORIZATION_REDIRECT_URI=https://你的域名/api/wechat/open-platform/authorization/callback
WECHAT_FRONTEND_SUCCESS_URL=https://你的域名/wechat/accounts
WECHAT_CREDENTIAL_ENCRYPTION_KEY=固定保存的Base64编码AES密钥
```

注意：

- 微信开放平台后台的授权事件接收 URL 配置为：`https://你的域名/api/wechat/open-platform/ticket`。
- `WECHAT_CREDENTIAL_ENCRYPTION_KEY` 必须固定备份；丢失后无法解密已保存的授权凭证。
- 所有 AppSecret、access token 和 refresh token 都只在后端使用，不会返回浏览器。

## 使用流程

1. 用户进入“公众号”页面，点击“授权公众号”，由公众号管理员扫码授权。
2. 确保文章状态为 `COMPLETED`，并且存在 `fullContent` 或 `content`。
3. 在文章详情页选择目标公众号。
4. 点击“保存到公众号草稿箱”，后端会创建公众号草稿并保存 `mediaId`。
5. 点击“发布到公众号”，后端会提交发布并保存 `publishId`。
6. 若已有 `publishId`，可点击“查询官方状态”刷新发布结果和文章链接。

## 状态说明

| 状态              | 含义       |
| --------------- | -------- |
| `DRAFT_CREATED` | 已创建公众号草稿 |
| `SUBMITTED`     | 已提交发布    |
| `PUBLISHING`    | 微信侧仍在发布中 |
| `SUCCESS`       | 发布成功     |
| `FAILED`        | 发布失败     |

## 后续建议

- 接入真实公众号前，先确认公众号已开通草稿箱和发布相关接口权限。
- 正式上线必须使用公网可访问的 HTTPS 域名，并在微信开放平台正确配置授权事件和授权回调域名。
- 如果生成正文中包含 SVG 图片，当前 MVP 会明确失败；后续可增加 SVG 转 PNG 后再上传微信。
- `frontend/src/api/articleController.ts` 和 `frontend/src/api/typings.d.ts` 是生成式 API 文件，后续重新执行 openapi 生成时需要保留本次新增的微信接口。
